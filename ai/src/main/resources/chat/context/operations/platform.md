# PLATFORM — OPENAT 플랫폼 개요

OPENAT은 굿즈와 한정판 상품을 정해진 시각에 정해진 수량만 판매하는 드롭 커머스 플랫폼이다. 드롭이 열리는 순간 한 상품에 주문이 몰려도 한정 수량보다 많이 팔지 않고, 결제 실패·취소·환불 뒤에는 주문과 재고 상태가 어긋나지 않게 복구하는 것이 핵심 목표다.

핵심 사용자는 상품과 드롭을 등록하는 판매자, 드롭 상품을 구매하는 회원, 전체 흐름과 장애를 확인하는 관리자다. 회원은 여러 판매자 정보를 가질 수 있으며 판매자 식별자와 회원 식별자는 서로 다르다.

주요 업무 흐름:

1. 판매자가 상품과 판매 시각·수량·사용자별 구매 한도를 가진 드롭을 등록한다.
2. 드롭이 열리면 상품 서비스가 Redis와 Lua로 재고 검사·차감을 원자적으로 처리하고 변경 이력을 영속 원장에 추가한다.
3. 주문 서비스가 주문을 만들고 재고 확보와 결제를 조율하는 사가 오케스트레이터 역할을 한다.
4. 결제 서비스가 지갑 또는 PG 결제, 취소와 환불을 처리하고 결과 이벤트를 발행한다.
5. 결제 완료 주문은 이벤트로 정산 서비스에 적재되며 일별 대사와 월별 판매자 정산의 근거가 된다.
6. 실패나 시간초과가 발생하면 주문 서비스가 재고 복원 같은 보상 트랜잭션을 수행한다.

플랫폼은 회원, 상품, 주문, 결제, 정산을 독립 서비스로 나눈 MSA다. 외부 요청은 API Gateway에서 JWT를 검증한 뒤 전달한다. 즉시 성공 여부가 필요한 재고 처리 등은 내부 동기 API를, 완료 뒤 전파하는 결제·환불·정산 데이터는 Kafka 이벤트를 사용한다. 분산 정합성은 2PC가 아니라 멱등 처리, 사가와 보상, outbox·inbox를 통해 맞춘다.

관리자 AI 어시스턴트는 읽기 전용 도구와 승인된 운영 문서를 조합해 자연어로 설명한다. 현재 운영 데이터를 임의로 수정하거나 장애 원인을 추측하지 않는다. 플랫폼 설명 질문에는 먼저 “한정 수량 드롭 커머스”라는 정체성과 사용자·전체 흐름을 설명하고, Redis 재고 차감은 이를 지탱하는 핵심 기술 중 하나로만 소개한다.
