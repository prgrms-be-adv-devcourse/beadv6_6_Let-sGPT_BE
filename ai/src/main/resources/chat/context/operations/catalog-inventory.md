# CATALOG_INVENTORY — 상품·드롭·재고

상품 서비스는 상품 마스터, 카테고리, 한정 판매 드롭과 재고를 소유한다. 상품은 설명·가격·이미지와 선택형 카테고리를 가질 수 있다. 카테고리가 없으면 미분류 상품이며 카테고리 삭제 시 참조 상품은 미분류로 전환된다. 상품은 soft delete를 사용하고 진행 중인 드롭이 연결돼 있으면 삭제가 제한된다.

드롭은 상품의 판매 기간, 총수량, 사용자별 구매 한도를 정의한다. 상태 이름은 `REGISTERED`, `OPEN`, `CLOSE`, `SOLD_OUT`이지만 DB에는 주로 영속 상태인 `REGISTERED`와 `CLOSE`를 저장한다. `OPEN`과 `SOLD_OUT`은 현재 시각과 잔여 재고를 반영한 파생 상태다. 따라서 DB 상태 하나만 보고 현재 판매 가능 여부를 판단하지 않는다.

드롭의 핵심 판정:

- 오픈 시각 전이면 `REGISTERED`
- 판매 기간 안이고 잔여 재고가 있으면 `OPEN`
- 잔여 재고가 0이면 `SOLD_OUT`
- 명시적으로 닫혔거나 종료 시각이 지났으면 `CLOSE`

재고의 실시간 게이트키퍼는 Redis와 Lua다. “드롭이 열렸는가, 재고가 남았는가, 사용자의 구매 한도를 넘는가”를 한 원자 연산에서 검사하고 차감한다. 영속 근거는 `stock_histories`에 부호 있는 `quantity_delta`로 추가되는 append-only 변경 이력이다. 주문별 변경 유형의 유일 제약이 중복 차감·중복 복원을 방어한다.

재고를 운영자가 DB에서 직접 수정하면 Redis 상태와 영속 원장이 어긋날 수 있다. 반드시 정식 차감·롤백 경로를 사용한다. 이상 징후는 드롭의 현재 파생 상태, 최초 총수량과 이력 합계, 음수 또는 최초 수량 초과 여부, 주문 보상 실패를 함께 확인한다.

현재 관리자 데이터 도구는 상품 수를 생명주기·카테고리별로, 드롭 수를 현재 상태·카테고리·재고 상태별로 집계한다. 공개 상품 ID·상품명·카테고리·현재 정상가와 공개 드롭 ID·상품명·판매가·재고·상태·일정은 비민감 운영 필드로 최대 20행까지 조회할 수 있다. 개별 가격이나 재고 값은 대응하는 공개 ID 차원이 있을 때만 실행한다. 판매자 정보와 내부 식별자는 결과에 포함하지 않는다. “현재 드롭”과 “특정 기간에 생성된 드롭”은 서로 다른 기준이므로 질문의 시간 기준을 구분한다.

운영자는 오픈 전 시간·수량·구매 한도 설정, 오픈 직후 품절과 주문 실패 변화, 종료 후 상태 전이, 미분류 또는 필수 정보가 빠진 상품을 확인한다. 집계만으로 개별 원인을 단정하지 않고 필요하면 주문·사가 결과와 교차 확인한다.
