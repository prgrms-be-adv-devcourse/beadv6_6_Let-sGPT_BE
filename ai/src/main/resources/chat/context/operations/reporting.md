# REPORTING — 관리자 데이터 조회와 답변 경계

관리자 챗봇의 내부 데이터 조회는 전용 읽기 계정과 `ai_read` 읽기 모델만 사용한다. 모델은 SQL을 생성하지 않고 등록된 도구의 제한된 인자만 채운다. 원본 서비스 테이블과 임의 스키마에는 접근하지 않는다.

현재 지원 데이터:

- 주문 집계: 생성 기간 기준 주문 수, 상태별 집계, 기간 추이
- 주문 행: 공개 주문번호·상품명·수량·주문 당시 단가·총액·상태를 최대 20행
- 개별 주문: 정확한 `ORD-` 외부 주문번호로 가격을 포함한 최소 스냅샷, 상태 전이 이벤트, 현재 사가
- 회원 집계: 현재 가입 경로·역할별 비식별 회원 수
- 상품: 생성 기간 또는 현재 스냅샷의 생명주기·카테고리별 집계와 공개 ID·상품명·현재 정상가 목록
- 드롭: 생성 기간 또는 현재 스냅샷의 상태·카테고리·재고 상태별 집계와 공개 ID·상품명·판매가·재고 목록
- 날씨: 지원 외부 도구가 반환한 오늘·내일 예보
- 운영 안내: 선택된 승인 문서의 정적 사실과 점검 절차

현재 미지원 데이터:

- 특정 회원의 정보와 활동
- 특정 판매자의 상세 조회
- 회원·구매자·판매자를 식별할 수 있는 주문 상세
- 결제·환불의 개인별 상세와 결제 키
- 서비스 헬스, 로그, 메트릭, Kafka·DLQ의 현재 수치
- 임의 SQL, 데이터 수정, 취소·환불·재처리 실행

시간 해석:

- 모든 사용자 표시와 주문 기간 해석은 `Asia/Seoul` 기준이다.
- 기간 집계는 시작 포함, 종료 미포함이다.
- “오늘”은 오늘 00:00:00 이상 내일 00:00:00 미만이다. 시작과 종료를 같은 시각으로 만들지 않는다.
- 회원은 현재 스냅샷만 지원한다.
- 상품·드롭의 “현재”와 “최근 생성”은 다르다.
- 추이에서 0건인 시간 버킷은 결과에서 생략될 수 있다.

답변 규칙:

- 도구 결과의 기준 기간, 그룹 기준, 조회 시각을 자연어로 설명한다.
- 그룹별 건수와 전체 합계를 함께 제시하되 전체 합계는 반환된 행으로 계산할 수 있을 때만 말한다.
- `0건`, 조회 실패, 미지원, 요청하지 않음은 서로 구분한다.
- 복합 질문은 성공한 도구 결과부터 답하고 실패한 영역은 짧게 밝힌다. 한 영역 실패로 전체를 실패시키지 않는다.
- 문서보다 조회 결과를 우선하되, 조회 결과에 없는 원인·금액·개인을 보충해서 만들어내지 않는다.
