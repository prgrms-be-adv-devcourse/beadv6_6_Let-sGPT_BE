# MEMBER_ACCESS — 회원·판매자·관리자

회원 서비스는 가입과 로그인, RS256 JWT 발급, 판매자 등록과 역할을 담당한다. 가입 경로는 `LOCAL`, `KAKAO`, `GOOGLE`, `NAVER`다. 역할은 `ROLE_USER`, `ROLE_SELLER`, `ROLE_ADMIN`이며 현재 역할 이력이 없으면 관리자 집계에서 `UNASSIGNED`로 표현될 수 있다.

회원 한 명은 여러 판매자 정보를 가질 수 있다. 회원 ID와 판매자 ID는 서로 다른 식별자다. 판매자가 상품을 관리할 때는 회원 로그인 토큰으로 선택한 판매자에 대한 scoped JWT를 발급받고, Gateway가 검증한 판매자 ID만 상품 서비스에 전달한다.

외부 요청은 API Gateway가 회원 서비스의 JWKS로 JWT를 검증한 뒤 신뢰 가능한 사용자 헤더를 다시 구성한다. `ADMIN` 역할은 관리자 챗봇과 관리자 API 접근의 기준이다. 다만 “내부 API”라는 이름 자체가 네트워크 보호를 보장하지 않으므로 실제 배포에서는 Gateway 우회 차단과 NetworkPolicy를 별도로 확인해야 한다.

관리자 챗봇이 제공하는 회원 데이터는 현재 활성 회원 수의 비식별 집계뿐이다. 가입 경로별, 현재 역할별 또는 둘의 조합으로 집계할 수 있다. 이메일, 전화번호, 주소, 개인별 활동, 특정 회원 조회, 작은 집단을 이용한 개인 역추적은 지원하지 않는다. 모델이 임의 SQL이나 회원 원본 행을 요청하도록 허용하지 않는다.

회원 집계는 “현재 스냅샷”이다. 주문처럼 과거 기간 추이를 의미하지 않으며 탈퇴 처리된 회원은 제외한다. 답변에는 현재 역할 기준인지 가입 경로 기준인지 밝히고, 특정 회원에 관한 질문은 가능한 비식별 집계로 바꿔 물어볼 수 있도록 안내한다.

운영 관점에서는 역할별·가입 경로별 분포의 급격한 변화, 역할 미할당, 판매자 역할 발급 흐름을 확인할 수 있다. 집계 변화만으로 가입 장애나 권한 오작동을 단정하지 않고 인증 로그와 서비스 상태를 추가 확인한다.
